# From ClickHelp to Docs-as-Code

Welcome to this beginner-friendly guide for traditional tech writers 
who want to transition to a docs-as-code workflow using MkDocs and 
GitHub Pages.

## What You Will Learn

- What docs-as-code is and why it matters
- How it compares to ClickHelp
- Setting up MkDocs step by step
- Publishing your docs on GitHub Pages

## Who Is This Guide For?

This guide is written for tech writers who:

- Use traditional tools like ClickHelp, MadCap Flare, or RoboHelp
- Have heard about docs-as-code but don't know where to start
- Want a real-world example to follow
