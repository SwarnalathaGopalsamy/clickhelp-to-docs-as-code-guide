# From ClickHelp to Docs-as-Code: A Beginner's Guide for Traditional Tech Writers

![License: CC BY 4.0](https://img.shields.io/badge/License-CC%20BY%204.0-lightgrey.svg)

## About This Project

This guide documents a real-world journey of transitioning from a 
traditional documentation tool (ClickHelp) to a docs-as-code workflow 
using MkDocs and GitHub Pages — written by a Senior Technical Writer, 
for technical writers.

If you have always used tools like ClickHelp, MadCap Flare, or 
RoboHelp, and have heard about docs-as-code but don't know where to 
start — this guide is for you.

## What You Will Learn

- What docs-as-code is and why it matters
- How docs-as-code compares to traditional tools like ClickHelp
- Setting up MkDocs step by step
- Publishing your docs on GitHub Pages
- A real-world example from a working portfolio
